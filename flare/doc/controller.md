# RpcController

RpcController 控制一次单独的 RPC 调用，并可设置一些调用相关的选项获得调用的附加信息。

客户端和服务器端分别使用 `RpcClientController` 和 `RpcServerController`，在不同的场景下，比如客户端发起调用前，调用返回后以及服务器端处理函数里，都只能使用其中特定的一些成员函数。

通过 RpcController，除了获取和设置调用错误外，还可以：

- 控制重试次数
- 控制超时
- 控制数据压缩方式
- 传输和获取额外的字节块数据
- 绕过框架自己进行序列化和反序列化
- 获得对方 IP 地址和端口
- 向分布式调用追踪报告更多的信息
- 获得调用耗时
- 获得调用期间各个阶段的时间戳

具体请阅读源代码中的注释。
